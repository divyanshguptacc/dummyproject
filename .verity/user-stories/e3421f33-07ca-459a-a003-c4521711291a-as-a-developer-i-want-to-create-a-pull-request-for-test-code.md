## Description

A developer needs the ability to create a pull request (PR) specifically for test code changes. This allows for proper code review, collaboration, and integration of test-related updates into the main codebase. Ensuring a streamlined PR process for test code helps maintain code quality and supports continuous integration workflows.

## Acceptance Criteria

- [ ] Given a feature branch containing test code changes, When the developer initiates a pull request, Then the system should allow creation of the PR targeting the main or designated branch.
- [ ] Given a created pull request for test code, When reviewers are assigned, Then they should be able to view the test code changes and leave comments.
- [ ] Given an open pull request for test code, When all required checks and reviews are completed, Then the PR should be eligible for merging into the main codebase.
